Project Overview

This project is a real-time ocean surface simulation driven by oceanographic spectra and Fast Fourier Transforms (FFT). Built from scratch without the use of commercial game engines, the primary goal was to master low-level graphics programming, custom rendering pipelines and GPGPU compute architecture.

Technical Architecture & Implementation

The simulation pipeline is built upon several advanced mathematical and rendering concepts:

  • Low-Level API: Engineered the custom rendering engine using DirectX 11 and C++ to ensure complete control over the graphics pipeline, memory management and shader execution.
  • Mathematical Foundations: Implemented J. Tessendorf's highly regarded Simulating Ocean Water paper to achieve physically plausible wave behaviors.
  • Oceanographic Spectra: Utilized the JONSWAP spectrum to dynamically generate initial ocean states in the frequency domain, directly controlled by physical parameters such as wind speed, fetch length and direction.
  • GPGPU Compute: Used Compute Shaders to execute the initial spectrum, Fast Fourier Transform (FFT) and displacement and slope calculation algorithms, highly parallelizing the calculation and conversion of frequency domain data into spatial heightmaps and normal maps in real-time.
  • Dynamic Tessellation: Implemented hardware tessellation with dynamic Level of Detail (LOD). The Hull and Domain shaders adaptively subdivide the ocean plane based on camera distance before displacing vertices using the FFT-generated textures.
  • Wave Cascading: Generated four independent ocean surface textures with varying patch sizes. Summing these cascades simulates a vast range of frequencies, from massive swells to high-frequency ripples, while effectively eliminating visual tiling artifacts in the distance.
  • Real-Time Controls: Integrated ImGui to allow real-time manipulation of simulation parameters, rasterizer states and lighting colors for immediate visual feedback.

By dynamically adjusting the JONSWAP spectrum parameters, such as wind speed and fetch length, the engine can simulate a vast range of sea states in real-time, from calm waters to turbulent, storm-driven swells.

Challenges Overcome

Building a fully custom ocean renderer from scratch presented several complex graphics programming hurdles:

  • Initial Spectrum Generation: Generating the initial spectrum involved solving complex equations where a single floating-point anomaly could corrupt the entire pipeline. A notable edge-case arose with the HLSL tanh function, where extremely high inputs produced NaN (Not-a-Number) values that poisoned the resulting textures. Leveraging RenderDoc to step through the texture generation allowed me to isolate the anomaly and patch the math errors safely.
  • T-Junctions & Mesh Tearing: Initially, I calculated the dynamic tessellation factor based on the distance from the camera to the center of each quad patch. This caused neighboring patches to evaluate different LOD levels, creating visible gaps (T-junctions) in the mesh. Shifting the tessellation calculation to evaluate the camera's distance from the patch edges ensured seamless LOD transitions and completely closed the mesh.
  • Tiling Artifacts: A single FFT grid mathematically repeats itself, creating noticeable, artificial tiling on the horizon. By combining four distinct cascades with prime-number patch sizes (2003m, 509m, 101m, and 23m), the repetition frequencies were desynchronized. This entirely masked the tiling, resulting in an infinite, non-repeating ocean surface.
  • Pipeline Debugging: The simulation relies on a sequential compute shader pipeline. A visual artifact in the final render could originate from an error in any intermediate pass. Integrating RenderDoc into the workflow was crucial; capturing frames and inspecting the Unordered Access Views (UAVs) at each stage allowed me to surgically identify which mathematical step of the pipeline was failing.

Future Roadmap & Next Steps

This simulation is an ongoing project. My immediate roadmap focuses on elevating the visual fidelity and expanding the engine to support interactive physics:

  • Physically Based Rendering (PBR): Currently, the ocean surface utilizes a basic Blinn-Phong reflection model. The next major milestone involves migrating to a fully Physically Based Rendering pipeline. This will include implementing BRDF and accurate Fresnel equations to properly blend deep-water subsurface scattering with real-time sky environment reflections.
  • Advanced Foam Rendering: Foam accumulation at wave peaks is currently calculated mathematically using the Jacobian determinant of the displacement map combined with exponential decay. While the underlying simulation logic works, the visual output requires refinement. Future iterations will focus on improving the foam blending pass so whitecaps look natural rather than painted on.
  • Interactive Buoyancy Physics: To transition this project from a purely visual simulation into a game-ready system, I plan to implement rigid-body buoyancy. This will require engineering a highly performant system to sample the exact wave height and surface normal at any given (X, Z) coordinate, allowing objects to interactively float and pitch with the waves.
  • Low-Level Optimization: Once the core feature set is fully implemented, the final phase will focus heavily on GPU profiling and optimization. I plan to rigorously profile and improve the Compute Shaders, refine the Hull Shader LOD logic and optimize the Domain and Pixel Shaders, ensuring the simulation runs well within strict real-time rendering budgets.

Core Implementation Highlight: Dispatching the Compute Shader

Generating the initial frequency spectrum requires binding specific constant buffers and Unordered Access Views (UAVs) to the pipeline immediately before dispatching the compute shader. Because the thread group size is defined as (16, 16, 1), the dispatch dimensions must dynamically adapt to fit the chosen texture resolution. This dispatch logic executes across all four cascades whenever the environmental simulation settings are modified.

void OceanComputeManager::GenerateInitialSpectrum(bool initial)
{
    RecalculateFrequencyFilters();

    ID3D11DeviceContext* context = D3D11Application::GetInstance().GetDeviceContext();

    context->CSSetShader(m_InitialSpectrumComputeShader, nullptr, 0);

    InitializeOceanSimulationSettings(initial);

    m_OceanSimulationSettingsBufferData.m_OceanTextureSize = m_OceanSimulationCascadeSettings.m_OceanTextureSize;

    // Calculate physical parameters
    m_OceanSimulationSettingsBufferData.m_PeakFrequency = pow(22.0f * m_OceanSimulationSettingsBufferData.m_GravitationalConstant * m_OceanSimulationSettingsBufferData.m_GravitationalConstant / max(m_OceanSimulationSettingsBufferData.m_AverageWindSpeed * m_OceanSimulationSettingsBufferData.m_FetchLength, 0.01f), 1.0f / 3.0f);
    m_OceanSimulationSettingsBufferData.m_Alpha = 0.076f * pow(m_OceanSimulationSettingsBufferData.m_AverageWindSpeed * m_OceanSimulationSettingsBufferData.m_AverageWindSpeed / (m_OceanSimulationSettingsBufferData.m_GravitationalConstant * m_OceanSimulationSettingsBufferData.m_FetchLength), 0.22f);
    m_OceanSimulationSettingsBufferData.m_WindAngle = m_OceanSimulationSettingsBufferData.m_WindDirection * (PI / 180.0f); // Convert to radians

    m_OceanSimulationSettingsBufferData.m_RandomSeed = rand();

    // Dispatch the compute shader for each cascade
    for (int i = 0; i < CASCADE_COUNT; i++)
    {
        m_OceanSimulationSettingsBufferData.m_PatchSize = m_OceanSimulationCascadeSettings.m_OceanPatchSize[i];
        m_OceanSimulationSettingsBufferData.m_LowPassFilter = m_LowPassFilters[i];
        m_OceanSimulationSettingsBufferData.m_HighPassFilter = m_HighPassFilters[i];
        m_OceanSimulationSettingsBufferData.m_CascadeAmplitude = m_OceanSimulationCascadeSettings.m_CascadeAmplitudes[i];

        context->UpdateSubresource(m_d3dOceanSimulationSettingsBuffer, 0, nullptr, &m_OceanSimulationSettingsBufferData, 0, 0);
        context->CSSetConstantBuffers(0, 1, &m_d3dOceanSimulationSettingsBuffer);

        context->CSSetUnorderedAccessViews(0, 1, &m_InitialSpectrumUAV[i], nullptr);

        // Adapt the dispatch groups to the texture resolution
        UINT groupsX = m_OceanSimulationCascadeSettings.m_OceanTextureSize / 16;
        UINT groupsY = m_OceanSimulationCascadeSettings.m_OceanTextureSize / 16;
        context->Dispatch(groupsX, groupsY, 1);

        // Unbind UAV to prevent read/write hazards
        ID3D11UnorderedAccessView* nullUAV[1] = { nullptr };
        context->CSSetUnorderedAccessViews(0, 1, nullUAV, nullptr);
    }
}
